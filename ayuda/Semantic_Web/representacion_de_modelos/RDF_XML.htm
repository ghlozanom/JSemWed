RDF/XML

Para serializar RDF y hacer posible el intercambio de aserciones entre aplicaciones y su posterior procesamiento, la W3C creo RDF/XML. RDF/XML define un lenguaje de etiquetado XML. Estas etiquetas permiten a los programas que las conozcan procesar la información contenida en el documento. El siguiente ejemplo muestra unas aserciones de un modelo de RDF en formato RDF/XML:

 

<?xml version="1.0"?>

<rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"

             xmlns:contact="http://www.w3.org/2000/10/swap/pim/contact#">

  <contact:Person rdf:about="http://proyecto-rg.tripod.com/participantes/contact#me">

    <contact:fullName>Gabriel Lozano</contact:fullName>

    <contact:mailbox rdf:resource="mailto:uno@hotmail.com"/>

    <contact:personalTitle>Mr.</contact:personalTitle>

  </contact:Person>

</rdf:RDF>

 

Este ejemplo dice que hay un contacto de tipo persona, que tiene nombre Gabriel Lozano, que tiene como correo uno@hotmail.com (en este caso no es cierto y esto ilustra que en Semantic Web cualquier persona o aplicación puede decir cualquier cosa sobre cualquier recurso, aspecto importante e interesante a tener en cuenta) y que tiene como titulo personal Mr. Este formato es fácilmente intercambiable entre aplicaciones y si la aplicación conoce estos tags, podrá sacar provecho de ellos. Por ejemplo, una aplicación de correo, puede obtener la dirección de correo de Gabriel Lozano para enviarle un mensaje importante a partir de estas aserciones.

Triples

En esta notación, cada aserción simplemente consiste de los URI del Sujeto y Predicado y del URI o valor literal del Objeto escritos en ese orden. Para escribir un URI se encierra entre corchetes y un valor literal se encierra entre comillas. Por ejemplo:

 

<http://proyecto-rg.tripod.../contacto#gl>  < http://www.w3.../contact#mailbox>  <mailto:uno@hotmail.com>

 

<http://proyecto-rg.tripod.../contacto#gl>  < http://www.w3.../contact#FullName> "Gabriel Lozano”

 

Esta notación puede resultar muy extensa y poco practica debido a la longitud de los URI de las partes. Para esto, se utilizan los XML qualified names o QNames. Un qname consiste de un prefijo asignado previamente a un URI, seguido por dos puntos (:) y un nombre local. Así, por ejemplo, si a la URI http://proyecto-rg.tripod.com/participantes/contacto# se le asigna el prefijo participantes, entonces el qname participantes:Gabriel es la abreviación de http://proyecto-rg.tripod.com/participantes/contacto#Gabriel. Así, las tripletas anteriores quedarían de la siguiente manera:

 

Prefijos:

participantes: URi: http://proyecto-rg.tripod.com/participantes/contacto#

contacto: URI: http://www.w3.org/2000/10/swap/pim/contact#

 

Aserciones:

participantes:gl contacto:mailbox mailto:uno@hotmail.com

participantes:gl contacto:FullName “Gabriel Lozano”

Diseño de Semantic Web

RDF es un lenguaje para decir cosas acerca de recursos. Este lenguaje esta diseñado para situaciones en las que los datos no solo deben ser enseñados o mostrados al usuario sino también usados por diferentes aplicaciones, así que debe ser posible serializarlo para ser procesado por máquinas.

 

RDF por sus objetivos tiene 2 necesidades de diseño importantes :

 

  • Un sistema que permita identificar de manera inequívoca las diferentes partes que componen las aserciones. Es decir, un sistema que permita identificar Sujeto, Objeto y Predicado sin lugar a equivocaciones.

  • Una serialización del modelo RDF que permita el intercambio entre aplicaciones de aserciones de tal forma que se puedan procesar.

 

Para identificar las diferentes partes que componen las aserciones, se utilizan URI, y para serializar el modelo W3C utiliza RDF/XML el cual define una serialización del modelo en XML.

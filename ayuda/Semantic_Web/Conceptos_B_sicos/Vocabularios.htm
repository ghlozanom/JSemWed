Vocabularios

En general, a un conjunto de URIs con un prefijo común (que normalmente es diseñado con un fin común, como, por ejemplo, describir documentos o elementos de una industria en particular) se le llama vocabulario. Típicamente, este URI que se utiliza como prefijo esta bajo el dominio del encargado del vocabulario. Así, por ejemplo, el vocabulario RDF tiene como URI común http://www.w3.org/1999/02/22-rdf-syntax-ns#. Este vocabulario define términos con un significado especial en RDF. En las aserciones se pueden utilizar cualquier número de vocabularios.

 

Por medio del uso de URIs en las aserciones RDF, se promueve el uso e intercambio de vocabularios en la Web. Esto expande el entendimiento común de las cosas. Así, una persona o aplicación puede utilizar un vocabulario desarrollado por otra persona u organización con conocimiento previo de su significado. Por ejemplo, Dublín Core Metadata Attribute Set es un vocabulario que se utiliza para describir documentos de todo tipo. Por ejemplo se puede utilizar dc:creator con dc como prefijo de Dublín Core Metadata Attribute Set como predicado para decir que cierta persona es la creadora de cierta página. Con el conocimiento que este Predicado sirve para esta tarea, una aplicación puede hacer uso de aserciones que tengan dc:creator como Predicado. Este significado no es inherente a la aserción, cualquier persona lo puede interpretar de cualquier manera. Por ejemplo, se puede crear un vocabulario y decir que ef:dfryu  sirve para decir lo mismo que dice dc:creator. Sin embargo, si una aplicación es escrita de manera correcta, es decir, aprovechando el significado que se le quería dar a un Predicado, parecerá entender el significado que traen las aserciones que haya podido obtener.

 

Por ejemplo, supongamos que existe un vocabulario conocido, bien hecho y aprobado por una organización rectora de la educación acerca de los estudiantes. Un vocabulario que describa sus notas de la mejor manera en conjunto con un vocabulario que los describa (nombre, edad, ubicación, estrato social, etc.). Cada colegio puede publicar la información educativa completa acerca de sus estudiantes, haciendo uso de estos vocabularios. Otra organización puede obtener toda esta información y organizar a los estudiantes o por un ranking general, o por los mejores por estrato social, o por los mejores por ubicación, o por los mejores por edad, etc. Con esto, una entidad, por ejemplo, encargada de otorgar becas a los estudiantes, podrá a partir de la información obtenida y compartida anteriormente, otorgar estas becas de una manera lo más justa posible. Todo esto lo hace posible un conocimiento y un entendimiento mutuo de vocabularios que permitan compartir toda esta información.

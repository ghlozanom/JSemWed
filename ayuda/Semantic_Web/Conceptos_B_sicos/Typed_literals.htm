Typed literals

A los literales que se utilizan como Objeto en las aserciones también se les puede asignar un tipo de datos que entregan mayor información sobre el literal y permitan su mejor procesamiento. A estos literales se les llama literales con tipo (typed literals, en ingles). Estos literales con tipo se usan cuando el valor de un literal se quiere utilizar como un tipo de valor especializado, como un número entero, un valor de fecha o un booleano, por ejemplo. Una aserción puede contener un Objeto que tenga por valor un número como, por ejemplo, el número de empleados con los que cuenta una organización, y una aplicación puede procesar perfectamente esta aserción, pero con un conocimiento previo del tipo de dato del Objeto de la aserción, no por información contenida en la aserción. Pero otra aplicación que no tenga este conocimiento previo no estaría en capacidad de procesar de manera correcta esta aserción.

 

Un literal de tipo en RDF se forma juntando el valor del literal con el URI que identifica al tipo del literal. Por ejemplo:

 

ej:presidencia ej:numero_emp “1200”^^xsd:integer

 

En este caso, el tipo de dato asociado al Objeto es xsd:integer.

 

Los tipos de datos utilizados son definidos externamente a RDF (con excepción de rdf:XMLLiteral, el cual se utiliza para identificar contenido XML en el Objeto). Por ejemplo, el tipo de datos xsd:integer es un tipo de dato de XMLSchema.
